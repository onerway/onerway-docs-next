All Onerway API requests require authentication using merchant credentials. When you create a Onerway account, we provide you with a complete set of credentials—merchant number, secret key, and store ID—for both sandbox and production environments. The **sandbox environment** lets you safely test Onerway features without processing real money, while the **production environment** handles live transactions.

## Before you begin

This guide walks you through a simple Onerway API interaction: creating a payment order. For a deeper dive into our API architecture and design principles, see the [API Overview](https://docs.onerway.com/apis/en/v0.6/){badge="API"}. If you're ready to start accepting real payments, check out our [Payments quickstart](mock/payments/quickstart).

## Manage your API credentials

Every account comes with complete [API credential sets](/get-started/start-building/start-developing#get-your-api-credentials) for both sandbox and production environments.

::warning
Can't see your API credentials? Ask your account owner to add you to their team with the appropriate permissions.
::

## Send your first API request

The easiest way to explore the Onerway API is through **[Apifox](https://apifox.com/)**, which lets you make API requests directly from your browser. Since it runs in sandbox mode, you can experiment without any risk to real funds.

1. Open the **[Card Checkout](https://apifox.onerway.com/api-265239300)** endpoint in Apifox
2. Click the `Run in Apifox` button
3. Use the preset sandbox environment variables (already configured for you)
4. Click **Send** to make the request

### Common error handling

You might encounter these common errors during testing:

| Error Code | Description | Handling Suggestion |
|------------|-------------|-------------------|
| `40013` | Abnormal parameters (cannot be read) | Check if environment variables are properly configured |
| `70002` | Unknown decline | Contact technical support |

::note
For more error codes, see [Error Code](https://docs.onerway.com/failure)

## View transaction data

Every payment order you create generates a complete transaction record. View and manage your transactions in the :prose-link-switch{preset="merchant-dashboard" path="/transaction/query" label="Onerway Dashboard"}.

### Transaction list

1. Access your :prose-link-switch{preset="merchant-dashboard" path="/transaction/query" label="Onerway Dashboard"}
2. Navigate to **Transactions** to see all payment activity
3. **Filter and search** by date, status, or other criteria

### Transaction details

Click any transaction ID to see detailed information across these sections:

::prose-tabs

  :::prose-tabs-item{label="Transaction Info"}
  - **Transaction ID**: Unique identifier for this transaction
  - **Transaction Type**: `Sale`, `Auth`, etc.
  - **Order Amount**: Original transaction amount
  - **Processing Time**: When the transaction was processed
  - **Response Code & Message**: Status and description from the payment processor
  :::

  :::prose-tabs-item{label="Payment Info"}
  - **Payment Type**: Card, local payment method, etc.
  - **Payment Mode**: Direct, subscription, or tokenized payment
  - **Card Brand**: Visa, Mastercard, etc.
  - **Issuing Country**: Where the card was issued
  - **Card Number**: Masked card number (e.g., `4111 11** **** 1111`)
  :::

  :::prose-tabs-item{label="3D Authentication"}
  - **3D Secure**: Whether additional authentication was required
  - **ECI**: Electronic Commerce Indicator
  :::

  :::prose-tabs-item{label="Risk Control Info"}
  - **Customer IP**: IP address used for the payment
  - **Device Info**: Browser and device details
  - **Risk Indicators**: Fraud and chargeback risk signals
  :::

  :::prose-tabs-item{label="Merchant Info"}
  - **Your Order ID**: Reference number from your system
  - **Application ID**: Which app processed this payment
  - **Origin URL**: Where the payment was initiated
  - **Product Details**: Item information and pricing
  - **Integration**: How the payment was processed
  :::

  :::prose-tabs-item{label="Customer Address"}
  - **Billing Address**: Customer's payment details and address
  - **Shipping Address**: Delivery information (when provided)
  :::

::

## After your payment

### Redirect customers to a return page

Customize where customers land after completing their payment.

#### Set up landing page

Add a `returnUrl` to your payment request:

::code-collapse

```json
{
  "txnOrderMsg": {
    // ...other fields...
    "returnUrl": "https://your-domain.com/payment/landing_page"
  }
}
```
::

::warning

Don't rely solely on `returnUrl` for payment confirmation. Customers might close their browser or lose internet connection before reaching your landing page.

[Set up a webhook event handler](mock/webhooks#set-up-webhook-handler) so Onerway can send payment events directly to your server, bypassing the client entirely. If webhook notification :prose-annotation[[delivery fails](/mock/webhooks)]{annotation="Merchant failed to respond correctly"}, Onerway will [retry multiple times](mock/webhooks#webhook-retry).

::

### Set up webhook handler

Webhooks notify your server about payment events in real-time, independent of the customer's browser.

#### Set up webhook endpoint

Add a `notifyUrl` to your payment request:

::code-collapse

```json
{
  "txnOrderMsg": {
    // ...other fields...
    "notifyUrl": "https://your-domain.com/webhook/onerway"
  }
}
```
::

#### Webhook handling

Learn how to process webhook events in our [Webhooks guide](https://docs.onerway.com/apis/en/v0.6/transaction-notification){badge="API"}.
