在编写代码之前，先了解端到端的集成流程。Onerway 在每个阶段提供专属支持：

- **安全测试** — 在隔离的 **Sandbox** 环境中运行功能测试和回归测试，不影响生产环境。
- **集中管理** — 在同一平台配置 **API 密钥**、支付方式、风控规则和订阅设置。
- **自动化事件** — 订阅异步回调事件，监听每个关键交易和订阅生命周期节点。

## 集成流程概览

集成 Onerway 支付方式时，按以下 7 个步骤进行：

1. **启动会议** — Onerway MI（Merchant Integration，商户集成）代表邀请参加接入前会议，确认所需支付方式和 API。
2. **阅读文档** — 开始阅读 [API 文档](https://docs.onerway.com/apis/)。
3. **技术会议** — Onerway MI 安排技术会议，解答开发团队的 API 问题。
4. **集成 API** — 使用 **Sandbox** 账户凭证进行集成。
5. **沙盒测试** — 根据测试用例执行测试。
6. **配置风控** — 确定风控（欺诈防护）策略。
7. **上线** — 完成生产环境验证。

## 接入前启动会议

Onerway 根据业务情况提供支付建议。会议内容包括：

- 确认所需支付方式
- 确定支付场景对应的 API
- 讨论时间表和集成范围

## 阅读 API 文档

在 [API 文档](https://docs.onerway.com/apis/) 中阅读启动会议确认的 API。记录使用问题，在技术会议中讨论。

## 技术会议解答 API 问题

此会议主要面向开发团队。Onerway 将：

- 详细介绍每个需要使用的 API
- 解答阅读文档时遇到的问题
- 提供集成最佳实践

## 集成 Onerway API

将 Onerway API 集成到网站或应用中。整个过程中，Onerway 提供支持，帮助完成集成。

::note
先在 **Sandbox** 环境完成所有集成工作，通过全部测试用例后再切换到生产环境。
::

## 在 Sandbox 中测试集成

集成完成后，根据业务场景进行测试，完成 Onerway 测试用例。

| 环境 | Base URL |
|------|----------|
| **Sandbox** | `https://sandbox-acq.onerway.com` |
| **Production** | `https://acq.onerway.com` |

## 配置欺诈防护规则

完成测试阶段后，可以在 Sandbox 中执行完整支付流程。上线前，根据业务场景与 Onerway 讨论风控策略。

## 使用生产凭证上线

将以下内容替换为**生产**环境账户凭证：

- API 请求 Base URL
- **商户号**
- **密钥**
- **应用 ID**

然后完成一笔真实支付验证。验证成功后，集成即可投入使用。

::warning
切勿在源代码中硬编码 API 凭证。使用环境变量或平台的密钥管理服务。
::
