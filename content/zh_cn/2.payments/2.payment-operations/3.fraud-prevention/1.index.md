向 :prose-annotation[卡组织]{annotation="处理特定卡品牌交易的组织，如 Visa、Mastercard 或 American Express。"} 提交银行卡付款进行授权时，Onerway 会将 CVC、邮编和账单街道地址提供给卡组织进行验证（前提是你在结账流程中收集了这些信息）。

发卡行会将这些信息与持卡人档案中的信息进行比对。如果信息不匹配，验证检查将失败。

CVC 或邮编检查失败可能表明该笔付款存在欺诈风险，因此在履行订单前请仔细审核。

~每个 [Charge](#){badge="TODO"} 对象的 `source` 属性中都包含[验证响应](#){badge="TODO"}。你也可以在 [Dashboard](#){badge="TODO"} 中查看付款时找到验证结果。~

如果未收集任何信息，发卡行将无法执行验证检查。为避免此问题，请确保每笔付款都收集 CVC、邮编和账单地址。验证检查的结果有助于提升欺诈活动的检测能力。

## 验证检查结果

| 检查类型 | 通过             | 失败         | 不可用       |
| -------- | ---------------- | ------------ | ------------ |
| CVC      | 已验证持卡人身份 | 潜在欺诈风险 | 未执行检查   |
| AVS      | 地址与记录匹配   | 地址不匹配   | 发卡行不支持 |

## 使用 CVC 检查验证持卡人身份

CVC（或 CVV）是印在银行卡上的三位或四位数字，只有实际持有该卡的持卡人才能获取。在银行卡支付方式中要求提供 CVC 值时（无论是在支付时还是将支付方式关联到客户资料时），发卡行会在交易授权过程中验证该值。

~在 [Dashboard](#){badge="TODO"} 中启用[规则](#){badge="TODO"}后~，未通过发卡行验证的付款将在多种场景下被拦截。这不会影响无法执行 CVC 检查或 CVC 检查不可用的付款。例如，Apple Pay 等 :prose-annotation[数字钱包]{annotation="一种非接触式支付方式，存储信用卡和借记卡等支付选项，让用户可以通过智能设备进行购物。"} 或 :prose-annotation[离线支付]{annotation="不涉及客户直接参与的付款，使用之前收集的支付信息完成。"} 不需要此检查。

商家不能存储 CVC 号码，因此欺诈者不太可能通过计算机漏洞获取 CVC，这使得 CVC 验证成为一种有效的防欺诈工具。然而，CVC 验证无法防范银行卡的物理盗窃，也无法防范安全性较差的计算机或网站带来的风险。

::tip
为最大化欺诈防护，请在每笔付款中收集 CVC。大多数发卡行会拦截 CVC 验证失败的付款。
::

## 使用 AVS 验证账单地址

在银行卡支付方式中要求提供邮编和账单街道地址时（无论是在支付时还是将支付方式关联到客户资料时），发卡行会执行 AVS 检查，以验证这些信息是否与档案中的账单地址匹配。

~在 [Dashboard](#){badge="TODO"} 中启用[规则](#){badge="TODO"}后~，未通过邮编（AVS）验证的付款将在多种场景下被拦截。合法付款的 AVS 检查也可能失败。例如，如果客户输入了错误的地址，或搬家后未通知发卡行，可能会看到地址检查失败的付款。

AVS 检查的支持情况因国家/地区和发卡行而异。例如，某些国家不使用邮编，某些发卡行不支持街道地址验证。在美国、加拿大和英国发行的大多数银行卡支持街道地址验证。
